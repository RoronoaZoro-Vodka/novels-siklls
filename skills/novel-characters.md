# 角色创作 (Character Creation)

用于高武小说角色体系的构建与管理。

## 1. 主角设定

### 基础人设
- 姓名、年龄、出身背景
- 外貌特征、标志性装扮
- 性格核心（至少3个关键词，如：坚韧、重情、不服输）
- 初始状态（天赋评价、实力起点、资源状况）

### 金手指/核心优势
- 天赋类型：修炼天赋、特殊体质、血脉传承
- 外挂介绍：系统、传承、重生记忆、特殊道具
- 限制条件：使用代价、时间限制、成长要求
- 进化路线：金手指本身也会升级/觉醒

### 成长弧规划
- 短期目标（当前卷/篇的追求）
- 中期目标（实力突破、势力建立）
- 长期目标（守护什么、达到什么境界）
- 关键转折点：每次重大突破的契机与代价
- 性格变化轨迹：从稚嫩到成熟的合理演进

### 主角的特色标签
- 招牌招式/技能（读者一眼识别的标志）
- 口头禅/标志性行为
- 战斗风格（狂猛/技巧/智取/诡道）

## 2. 核心配角团

每个核心配角需要：

### 基础档案
- 姓名、性别、年龄
- 外貌特征
- 实力等级与修炼路线
- 身份背景

### 与主角的关系定位
| 类型 | 举例 |
|------|------|
| 导师 | 引路人、授业恩师 |
| 战友 | 并肩作战、生死之交 |
| 对手 | 亦敌亦友、良性竞争 |
| 红颜/蓝颜 | 感情线对象 |
| 下属/追随者 | 忠诚、个人动机 |

### 独立性设计
- 自己的目标与追求（不只是主角的工具人）
- 高光时刻（独当一面的机会）
- 可能黑化/背叛的伏笔
- 如果死亡，需要什么意义

### ⚠️ OOC 红线：角色行为不得偏离人设基线
这是最容易踩坑的地方，也是最容易让读者出戏的原因。这个坑的本会话中真实踩过。

#### 真实案例——林雨薇（内向安静型角色）
- 设定：话不多，存在感低，成绩好但不出头
- ❌ 初期写了她主动拦住主角并谈论敏感话题（赵星河被带走）
- 用户指出：「林雨薇的表现确实有些不合理」
- ✅ 修正：删除该场景。内向型角色不会主动制造这类对话

#### 教训
如果一段对话或行为让作者本人都觉得「ta不像会说这种话的人」，就已经OOC了。这个信号不可忽略。

#### 常见OOC陷阱
- **内向/话少的角色突然主动找主角聊天**——除非有充分的动机铺垫（如被他人委托、发现关键信息等），否则不符合人设
- **冷静稳重的角色在危机中尖叫或慌乱**——与该角色的过往一致行为矛盾
- **高傲的角色低声下气求人**——除非经历了重大打击或转变，且这种转变应花篇幅去写，而不是一笔带过
- **为推进剧情让角色做不符合性格的事**——宁可删掉这个剧情点，也不要牺牲角色一致性

#### 核心原则
一个角色的言行，必须从头到尾由其性格驱动，而非由剧情需求驱动。
如果读者会想「ta不像会说这种话的人」，说明OOC了。

## 3. 反派体系

### 反派层次
- **地表级**：当前卷/篇章的对手
- **主线级**：贯穿大半个故事的反派
- **幕后级**：真正操控一切的黑手
- **最终BOSS**：世界级威胁

### 反派塑造
- 动机充分（不是为坏而坏）
- 魅力特点（实力、智慧、原则、过去）
- 与主角的渊源（宿命感）
- 洗白/退场的合理方式

## 4. 关系网管理

用表格记录角色之间的关系：

```markdown
| 角色A | 关系类型 | 角色B | 亲密度 | 备注 |
|-------|---------|-------|--------|------|
| [主角] | 师徒 | [导师] | ★★★★★ | 已拜师 |
| [主角] | 宿敌 | [反派] | ★☆☆☆☆ | 杀父之仇 |
```

每次新增重要互动后更新关系状态。

## 5. 角色进化追踪

每完成一个剧情大段，更新角色状态：

```markdown
## [角色名] 进化记录
- 初始境界: 武者三段
- 当前境界: 武师七段
- 重大突破: [事件] → 突破至 [境界]
- 新增技能: [技能名称与效果]
- 性格变化: [变化描述与原因]
- 关系变化: [新增/升级/破裂的关系]
```

## 6. 写作中易犯的角色Bug

### 外貌矛盾
- 同一角色的外貌描述不能自相矛盾
- 常见错误：同时写「扎着马尾」和「短发」（马尾必须有一定长度）
- 解决方案：每次让角色出场时，对照角色档案的外貌描述复核

### 觉醒状态矛盾
- 已觉醒的角色不应出现在「等待检测觉醒」的场景中
- 已有明确等级的角色不应被问及「检测准备」——他不需要准备，他已经知道自己的水平
- 解决方案：写前确认该角色在当前剧情时间点上的觉醒状态

### 名字使用统一
- 全名首次出现后用「他」或单名（江寻）交替，不要每句都用全名
- 从主角视角出发时，父亲角色用「他爸」而非全名「江镇山」
- 避免多个角色在同一段落中频繁使用全名造成混乱

### 检测类场景通用陷阱
- 如果是全校统一检测，已觉醒者也参加（测等级提升）
- 如果是「觉醒筛查」则已觉醒者不参加
- 在worldbuilding中明确检测机制，全篇保持一致
