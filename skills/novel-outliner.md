# 大纲架构 (Outline Architecture)

用于长篇高武小说的整体结构规划。

## 1. 主线架构

### 核心主线
一句话总结整个故事。

### 三条线并行
- **实力线**：主角从弱到强的成长轨迹，每一次突破
- **事件线**：世界格局变化、重大危机、势力洗牌
- **感情线**：友情/爱情/亲情的推进与高潮

### 全篇三幕结构
| 幕 | 阶段 | 主要内容 | 情绪基调 |
|----|------|---------|---------|
| 第一幕 | 崛起篇 | 入世、初露锋芒 | 热血、激情 |
| 第二幕 | 风暴篇 | 卷入大局、危机四伏 | 激烈、悲壮 |
| 第三幕 | 终局篇 | 终极决战、圆满收官 | 史诗、感动 |

## 2. 分卷规划

每卷需要明确：

```markdown
## 第X卷：[卷名]
- 字数：约[X]万字
- 核心冲突：[冲突描述]
- 主角突破：[从X境到Y境]
- 关键事件：[重要剧情节点]
- 新增角色：[重要新出场]
- 情感高潮：[感人/燃点情节]
- 结尾悬念：[钩子，吸引读下一卷]
```

### 卷间节奏
- 小卷（3-5万）：快速推进，适合前期
- 中卷（8-12万）：深入展开，适合发展期
- 大卷（15-20万）：宏大叙事，适合后期高潮
- 每卷结尾必须有悬念或情绪释放点

## 3. 章节节奏

### 黄金章节公式
```
开头钩子(1-2段) → 推进剧情 → 冲突/转折 → 爽点释放 → 结尾悬念
```

### 每章内容分配
- 对话占 30-40%
- 描写占 20-30%
- 剧情推进占 30-40%
- 保持信息密度，拒绝水字数

### 爽点分布
- **小爽点**：每1-2章至少一个（突破小境界、打赢同级）
- **中爽点**：每卷至少2-3个（越级战斗、获得宝物、关键胜利）
- **大爽点**：每2-3卷一个（大境界突破、斩杀宿敌、势力飞跃）
- **终极爽点**：全书高潮（最终决战、巅峰时刻）

## 4. 伏笔管理

```markdown
| 伏笔 | 埋下章节 | 回收章节 | 类型 | 状态 |
|------|---------|---------|------|------|
| [内容] | 第X章 | 第X章 | 剧情/实力/身份 | 未回收 |
```

## 5. 字数预估

| 部分 | 建议字数 | 说明 |
|------|---------|------|
| 开局 | 2-5万 | 世界观展示，第一个高潮 |
| 主体 | 200-500万 | 多卷展开 |
| 结局 | 10-20万 | 最终决战+尾声 |
| 全书 | 约250-500万 | 高武小说常见篇幅 |
